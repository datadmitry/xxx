Class Journal

Веб-платформа для учителей и учеников: создание тестов, отправка их классам, прохождение, ежедневные квизы, статистика и аналитика по классам.

введение

Идея и решаемые задачи

Проблема

  • Учителю неудобно вести разные тесты для разных классов вручную
  • Ученики теряют интерес без регулярной активности и обратной связи
  • Нет единого места, где видно успеваемость и прогресс

Решение — Class Journal

  • Общая база тестов с возможностью отправки в конкретный класс
  • Ежедневный квиз со счётчиком серии для удержания учеников
  • Личная статистика для ученика и аналитика классов для учителя
функциональность

Что умеет приложение

Учитель

  • Создаёт классы, к которым прикрепляются ученики
  • Пополняет общую базу тестов через «Создать тест»
  • Отправляет любой тест из базы или свой в выбранный класс
  • Делает предпросмотр теста с правильными ответами
  • Видит результаты и графики по каждому классу
  • Использует умный поиск по тестам (регистр и кириллица)

Ученик

  • Видит тесты, отправленные именно его классу
  • Проходит тест с живым таймером прохождения
  • Получает оценку 2–5 и подробный разбор ошибок
  • Решает ежедневный квиз и продлевает серию (огонёк)
  • Открывает «Мои результаты» с агрегированной статистикой

база данных · orm

ORM-классы (SQLAlchemy)

6 моделей

  • User — учитель или ученик, роль, уровень, серия квизов
  • Class — учебный класс (5А, 6Б …), привязан к учителю
  • Test — тест: либо в общей базе, либо отправлен классу
  • Question — вопрос теста: текст, изображение, правильный ответ
  • TestResult — результат прохождения учеником: балл, оценка, время
  • StudentAnswer — каждый ответ ученика и его правильность

Пример модели

class User(UserMixin, db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(100), unique=True)
    password = db.Column(db.String(200))
    role     = db.Column(db.String(20))
    level    = db.Column(db.Integer, default=1)
    class_level  = db.Column(db.Integer)
    class_letter = db.Column(db.String(1))
    teacher_code = db.Column(db.String(50))
    daily_streak    = db.Column(db.Integer, default=0)
    last_daily_date = db.Column(db.Date)
    test_results = db.relationship('TestResult',
                                   backref='student')

Идея: тест с class_id = NULL — это запись в общей базе тестов. Когда учитель «отправляет» тест классу, в БД создаётся новый Test с class_id + копии всех его Question.

файлы

Загрузка и использование файлов

Загрузка картинки к вопросу

  • FileField + FileAllowed(['jpg','png','jpeg']) — пускаем только нужные расширения
  • secure_filename из Werkzeug — защита от path traversal
  • Файл сохраняется в UPLOAD_FOLDER, в БД хранится только имя
  • Имя приводится к виду {{title}}{{q_num}}.png для удобной выдачи

Раздача файлов

@app.route('/uploads/<path:filename>')
def uploaded_file(filename):
    return send_from_directory(
        app.config['UPLOAD_FOLDER'],
        filename
    )

Шаблоны строят URL через хелпер question_image_url(name), поэтому путь к картинкам не зависит от того, локально мы или на Vercel.

Как это работает

  • api/index.py — точка входа, импортирует app из main
  • Vercel ставит зависимости из requirements.txt
  • Статика отдаётся CDN-ом, всё остальное — Python-функцией
  • Чтение из ветки main — каждый push автоматически деплоится
  • На сервере БД и загрузки лежат в /tmp/classjournal/ (только там можно писать)
  • Секрет приложения SECRET_KEY и DATABASE_URL можно задать через env Vercel
фишки

Что делает проект «оригинальным»

База тестов + отправка

Один Test с class_id IS NULL = запись в банке. «Отправить классу» = клонировать его и все вопросы под нужный class_id. Это позволяет переиспользовать тесты и хранить отдельные результаты для каждого класса.

Ежедневный квиз и серия 🔥

Один вопрос в день, last_daily_date и daily_streak в модели User. Серия растёт, если решено вчера и сегодня; иначе сбрасывается. Огонёк рядом с именем мотивирует возвращаться.

Умный поиск (RU)

SQLite LOWER() не понижает кириллицу. Поэтому фильтрация делается в Python: query.casefold() in title.casefold(). Учитель находит «Математика» по «матем», «МАТЕМ» и «Матем».

Премиум UI · Three.js

На главной — WebGL-сцена с плавающими школьными фигурами в брендовых цветах. Камера слегка реагирует на курсор — лёгкий, не перегруженный hero.

Предпросмотр и аналитика

Учитель видит правильные ответы перед отправкой. На странице класса — графики Chart.js со средним баллом и динамикой по тестам.

Личная статистика ученика

«Мои результаты»: количество, средний балл, лучший результат и общее время в виде stat-карточек + таблица истории прохождений с временем по каждому тесту.
